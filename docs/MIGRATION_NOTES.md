# 模型遷移與版本調整

> **用途**：整理 Gemini 模型遷移相關策略與結論。  
> **更新日期**：2026年1月15日

---

## 1) 遷移背景

- 主生成模型由 1.5 Flash 遷移至 2.0 Flash  
- 目標：降低 429、提高題目品質、維持成本  

---

## 2) 發現與重點

- API 呼叫層缺乏統一重試機制  
- 速率限制僅在前端，容易被預載/並發放大  
- Prompt 可強化（步驟推理、格式一致性）  

---

## 3) 建議方向

- 在 API 路由層加入重試與退避  
- 調整預載策略，避免併發過高  
- 保持模型名稱在 `constants.js` 統一管理
